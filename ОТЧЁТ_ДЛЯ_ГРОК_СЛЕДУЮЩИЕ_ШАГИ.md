## Следующие шаги (по порядку)

### Шаг 1. Деплой на Vercel

1. Зайти на **https://vercel.com**, авторизоваться (лучше через GitHub).
2. **Add New** → **Project** → импортировать репозиторий **ozharov164-glitch/cozyreset-path-to-self-miniapp**.
3. **Configure Project:**
   - **Framework Preset:** Vite  
   - **Build Command:** `npm run build`  
   - **Output Directory:** `dist`  
   - **Environment Variables:** добавить переменную  
     - **Name:** `VITE_BOT_BACKEND_URL`  
     - **Value:** URL бэкенда бота (например `https://your-backend.com` или `http://217.114.11.97:8080`), **без слэша в конце**.
4. Нажать **Deploy**. Дождаться сборки.
5. После деплоя скопировать **Production URL** (например `https://cozyreset-path-to-self-miniapp-xxx.vercel.app`) — это и есть **публичный URL Mini App**.

### Шаг 2. Подключить Mini App к боту

1. В коде бота (или в BotFather) задать **Web App URL** для кнопки/команды, открывающей Mini App: подставить полученный Vercel URL.
2. Убедиться, что бот открывает Mini App через `WebAppInfo(url=...)` или аналог, а не через `?start=...` как основной поток.

### Шаг 3. Реализовать эндпоинты на бэкенде бота

Mini App уже вызывает эти эндпоинты. Бэкенд должен их реализовать (если ещё не сделано):

| Метод | Путь | Назначение |
|-------|------|------------|
| POST | `/mini-app/init` | Уже есть. Тело: `{ "initData": "<string>" }`. Ответ: `{ "app_save_token": "<string>" }`. |
| POST | `/mini-app/save-test-result` | Сохранить результат теста. Тело: `testId`, `testTitle`, `answers` (массив 1–10), `dimensions?`, `completedAt`. В теле или заголовке передаётся `token` (app_save_token). Ответ: `{ "id": "<string>" }`. |
| POST | `/mini-app/test-history` | Список сохранённых результатов пользователя. Тело: `{}` или `{ "token": "<app_save_token>" }`. Ответ: `{ "items": [ { "id", "testId", "testTitle", "completedAt" } ] }`. |
| GET | `/mini-app/test-result/{id}` | Один результат по `id`. Query: `?token=<app_save_token>`. Ответ: `{ "id", "testId", "testTitle", "answers", "dimensions?", "completedAt" }`. |

- Авторизация: по `app_save_token` (в теле POST или в query GET) определить `user_id` (как в существующей логике бота).
- CORS: разрешить origin Vercel (и при необходимости `https://web.telegram.org`).

Точные форматы запросов/ответов — в **README.md** в корне проекта.

### Шаг 4. Проверка в Telegram

1. Открыть бота в Telegram.
2. Запустить Mini App по кнопке/команде.
3. Проверить: приветствие, каталог 7 тестов, прохождение теста (шкала 1–10), сохранение результата, экран «Результат сохранён ✅ Перейди в бота» и кнопку «Открыть бота», историю и просмотр сохранённого результата.
4. При 401 (если специально проверить) — приложение должно молча сделать re-init без показа ошибки пользователю.

## Важные детали

- **Шкала ответов в тестах:** 1–10 (не 1–5). В README в описании save-test-result может быть упомянуто «1–5» — считать устаревшим, везде 1–10.
- **Авторизация:** приоритет — токен из store → токен из hash (`#s=...` или `#token=...`) → `POST /mini-app/init` с initData из Telegram. Все запросы — с токеном; при 401 — один автоматический re-init.
- **После сохранения результата:** показывать «Результат сохранён ✅ Перейди в бота» и кнопку «Открыть бота» → `Telegram.WebApp.openTelegramLink('https://t.me/CozyReset_bot')`. Не использовать `sendData` как основной способ связи с ботом.
- **Переменная окружения:** только `VITE_BOT_BACKEND_URL` (без слэша в конце). При сборке на Vercel она подставляется в клиентский код.

## Если что-то пошло не так

- **Сборка падает на Vercel:** проверить, что в настройках проекта Build Command = `npm run build`, Output Directory = `dist`, установлены зависимости (`npm install` выполняется по умолчанию).
- **Mini App не открывается в Telegram:** проверить, что URL в боте совпадает с Production URL Vercel (https, без лишних путей).
- **401 при запросах:** проверить, что бэкенд принимает `app_save_token` (из init) и возвращает 401 только при невалидном/просроченном токене; в Mini App при 401 выполняется повторный init.
- **CORS:** на бэкенде разрешить origin Vercel и при необходимости `https://web.telegram.org`.

После выполнения шагов 1–4 Mini App «Путь к Себе» полностью готов к использованию в боте.
