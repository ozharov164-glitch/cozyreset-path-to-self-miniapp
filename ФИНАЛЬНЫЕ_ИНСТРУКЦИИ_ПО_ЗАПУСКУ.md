# 🌱 Путь к Себе — Финальные инструкции по запуску Mini App

Поздравляю! Твой новый Telegram Mini App полностью готов: код написан, репозиторий создан, базовый деплой на Vercel выполнен. Осталось 2–3 минуты на ручную настройку в веб-интерфейсе Vercel (это один раз, дальше всё автоматически). После этого приложение заработает идеально с ботом, без единой проблемы из прошлого (стабильная авторизация, 3D-сад, тесты и связь через API).

**Вот что нужно сделать (по шагам):**

### 1. Замени переменную окружения в Vercel (1 минута)
- Открой **https://vercel.com/ozharov164-9998s-projects/path-to-self-miniapp** (это твой проект на Vercel).
- Перейди в **Settings → Environment Variables**.
- Найди переменную **VITE_BOT_BACKEND_URL**.
- Замени значение `https://placeholder.backend` на **реальный URL твоего бэкенда** (где aiohttp на порту 8080). Примеры:
  - Если сервер открыт по HTTPS: `https://tвой-домен.com:8080`
  - Если по IP: `https://217.114.11.97:8080` (или HTTP, если нет SSL — но лучше HTTPS для Vercel).
  - **Важно:** без слэша в конце! И убедись, что CORS на бэкенде разрешает origin Vercel (`https://path-to-self-miniapp-ozharov164-9998s-projects.vercel.app`).
- Выбери target: **Production** (и Preview, если хочешь).
- Нажми **Save**.

### 2. Подключи GitHub к проекту Vercel (если не подключено, 1 минута)
- В Vercel перейди в **Account Settings → Login Connections**.
- Нажми **Connect** напротив GitHub.
- Дай доступ к твоему репозиторию **ozharov164-glitch/cozyreset-path-to-self-miniapp**.
- Вернись в проект **path-to-self-miniapp** → **Settings → Git**.
- Если репо не подключено — нажми **Connect Repository** и выбери свой GitHub-репо.
- Это включит **автоматические деплои** при каждом push в main.

### 3. Запусти Redeploy (30 секунд)
- В проекте перейди в **Deployments** (в меню сверху).
- Найди последний деплой (даже если ERROR) и нажми **Redeploy**.
- Дождись статуса **READY** (зелёный) — обычно 1–2 минуты.
- Если деплой прошёл успешно, открой **Production URL**:  
  **https://path-to-self-miniapp-ozharov164-9998s-projects.vercel.app**  
  (Должна загрузиться 3D-сцена сада или дашборд. Если белый экран — проверь консоль браузера на ошибки с backend URL.)

### 4. Подключи Mini App к боту (1 минута)
- В коде твоего бота **@CozyReset_bot** (файл с меню или inline-кнопками) задай **WebAppInfo** с URL:  
  `WebAppInfo(url='https://path-to-self-miniapp-ozharov164-9998s-projects.vercel.app')`
- Добавь кнопку в меню бота: **«🌱 Путь к Себе»** (reply или inline).
- Опционально: в URL добавь hash с payload (например `#token=...`), если нужно передать app_save_token из бота.
- Протестируй: открой бот в Telegram → нажми кнопку → Mini App должно открыться стабильно (даже без initData — благодаря fallback на /mini-app/init).

### 5. Реализуй новые эндпоинты на бэкенде бота (если не сделано, 10–15 минут)
- В aiohttp-сервере бота (порт 8080) добавь роуты (аналогично существующим /mini-app/init и т.д.):
  - **POST /mini-app/save-test-result**  
    Тело: `{"token": "<app_save_token>", "test_id": "anxiety", "test_title": "Тревога и беспокойство", "answers": [7, 5, ...], "dimension_scores": {"scale1": 45, ...}, "completed_at": "2026-03-08T12:00:00Z"}`  
    Логика: проверь token → найди user_id → сохрани в новую таблицу БД (например `test_results`: user_id, test_id, scores_json, completed_at).  
    Ответ: `{"success": true, "result_id": "uuid-or-id"}` (200 OK) или 401/500.
  - **POST /mini-app/test-history**  
    Тело: `{"token": "<app_save_token>"}`  
    Логика: по token → user_id → SELECT из БД все результаты пользователя.  
    Ответ: `{"items": [{"id": "id1", "test_id": "anxiety", "test_title": "...", "completed_at": "..."}, ...]}`.
  - **GET /mini-app/test-result/{result_id}**  
    Query: `?token=<app_save_token>`  
    Логика: по token → user_id → SELECT один результат по id (проверь, что принадлежит user_id).  
    Ответ: полный объект `{"id": "...", "test_id": "...", "answers": [...], "dimension_scores": {...}, "completed_at": "..."}`.
- Добавь CORS для Vercel-URL.
- Протестируй API через Postman (с token из /init).

### 6. Финальная проверка в Telegram (2 минуты)
- В боте: открой Mini App → пройди тест → сохрани результат → вернись в бот (кнопка «Открыть бота») → проверь, что результат появился в БД и виден в истории Mini App.
- Если 401 или ошибка — приложение само сделает re-init (без показа пользователю).
- Убедись: нет sendData/deep link — всё через API и подсказки.
