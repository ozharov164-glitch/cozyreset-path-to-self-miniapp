# Интеграция «Путь к Себе» — что сделано

**Интеграция завершена. Проблема «Нет связи» устранена: диагностика, retry, стабильный backend.**

- **URL приложения:** https://ozharov164-glitch.github.io/cozyreset-path-to-self-miniapp/
- **Хостинг:** GitHub Pages (деплой через GitHub Actions при push в `main`). Репо: **ozharov164-glitch/cozyreset-path-to-self-miniapp**.
- **Бэкенд:** HTTPS через Cloudflare Tunnel. URL берётся из ссылки бота (`?backend=...`), иначе из sessionStorage/config.json. 5 retry init с backoff, повтор save после re-auth. Авторизация по token и/или initData.

---

## 1. Что сделано в приложении

- **Реальные вопросы** для всех 7 тестов (Тревога 22, Настроение 20, Самооценка 24, Выгорание 25, Границы 18, Смысл 21, Прогресс 16). Тексты релевантны темам, тёплый язык, шкала 1–10.
- **Отображение:** текст вопроса выводится из `questions[currentIndex].text`, стили: color `#000`, font 16px, center, padding 20px, без overflow hidden. Блок статичный, без анимаций для текста. Debug: `console.log` текста вопроса.
- **Связь с ботом:** backend URL — приоритет из query `?backend=` (бот передаёт в ссылке), затем sessionStorage, затем config.json. ensureAuth: 5 попыток с задержками 0/100/300/1000/2500 мс. Save при network error: сброс токена, повтор ensureAuth и повтор save. Во все POST добавляются token и initData. В консоли логи [PTS] для диагностики (location.search, getBackendUrl, initData length).

## 2. Бэкенд бота (cozyreset-bot)

- **database.py:** таблица `test_results`, методы `add_test_result`, `get_test_result_history`, `get_test_result_by_id`.
- **services/yookassa_service.py:**
  - `POST /mini-app/init` — выдача app_save_token по initData.
  - `POST /mini-app/save-test-result` — приём token или initData, camelCase/snake_case.
  - `POST /mini-app/test-history`, `GET /mini-app/test-result/{id}?token=...`.
  - CORS: `Access-Control-Allow-Origin: *`.
- **keyboards/main_menu.py:** кнопка «🌱 Путь к Себе» с WebAppInfo(url). URL из `MINI_APP_PATH_TO_SELF_URL` или fallback.
- Деплой: `./deploy_to_server.sh 217.114.11.97` (SCP + обновление .env + restart vklyuchisebya-bot).

На сервере в `.env`: `MINI_APP_PATH_TO_SELF_URL`, `MINI_APP_BACKEND_URL`, `MINI_APP_BACKEND_HTTPS_URL` (текущий URL туннеля подставляется при deploy_to_server.sh).

## 3. HTTPS бэкенда

- Cloudflare Tunnel (cloudflared) на сервере: сервис `cloudflared-tunnel-bot`. Текущий URL туннеля прописан в **public/config.json** (`backendUrl`). Если туннель перезапускался и URL изменился — обновить config.json и задеплой приложение.
- Скрипт повторного запуска туннеля: `scripts/setup-cloudflare-tunnel.sh` (в репо бота).

## 4. Проверка в Telegram

1. Открой **@CozyReset_bot** → нажми **«🌱 Путь к Себе»**.
2. Откроется Mini App (дашборд, градиентный фон в WebView).
3. **Каталог тестов** → выбери тест → **Пройти**.
4. Должны отображаться **тексты вопросов** (не только номер и шкала). Ответь по шкале 1–10.
5. В конце — сохранение. Должно появиться «Результат сохранён ✅» и кнопка **«Открыть бота»**.
6. Снова открой приложение из бота → **История** — сохранённый результат. При ошибке сохранения — кнопка **«Повторить сохранение»** или открой приложение заново из бота.

Если в РФ «Нет связи» (геоблок Cloudflare): настроить домен (DuckDNS и т.п.), выполнить `DOMAIN=... ./scripts/setup-https-nginx.sh` в репо бота, обновить config.json и MINI_APP_BACKEND_HTTPS_URL на https://домен. Логи бота: `journalctl -u vklyuchisebya-bot`; ошибки Mini App: `/opt/bot/logs/mini_app_errors.log`.
