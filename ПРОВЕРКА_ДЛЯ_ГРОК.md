# Файл для проверки Гроку — Mini App «Путь к Себе»

Используй этот документ, чтобы проверить соответствие кода дизайн-спеку и требованиям. Дизайн-спек: `../MINI_APP_PUT_K_SEBE_DESIGN_SPEC.md` (в родительской папке `cozyreset-bot`).

---

## Что это

Telegram Mini App **«🌱 Путь к Себе»** (Сад Внутреннего Я) для бота CozyReset. React 18 + TypeScript + Vite, объёмные тесты (15–25 вопросов), сохранение результатов, история.

---

## Чеклист для проверки

### 1. Стек и зависимости
- [ ] Vite, React 18, TypeScript
- [ ] @react-three/fiber, @react-three/drei, three — установлены; полный 3D-сад «Сад Внутреннего Я» реализован (сцена, bloom, частицы, объекты по тестам)
- [ ] Tailwind (v4), Framer Motion, TanStack Query, Zustand, Recharts, Zod
- [ ] Telegram WebApp SDK (скрипт в `index.html`)

### 2. Авторизация (критично)
- [ ] Приоритет: `Telegram.WebApp.initDataUnsafe` → токен из hash URL (`#s=...` или `#token=...`) → `POST /mini-app/init`
- [ ] Токен `app_save_token` сохраняется в Zustand + localStorage (`src/store/authStore.ts`)
- [ ] Все запросы к API идут с токеном (в теле POST или в query для GET)
- [ ] При 401 — очистка токена и один автоматический re-init без показа ошибки пользователю
- [ ] Не используется `sendData` и не `openTelegramLink(?start=...)` как основной способ связи с ботом

### 3. Экраны и навигация
- [ ] **Dashboard** — приветствие, кнопки «Каталог тестов» и «История»
- [ ] **Catalog** — 7 тестов с названием, описанием (2–3 предложения), числом вопросов, кнопка «Пройти»
- [ ] **TestFlow** — по одному вопросу на экран, шкала 1–10, прогресс-бар, «Назад»; после последнего ответа → Result
- [ ] **Result** — после прохождения: сохранение через API, текст «Результат сохранён ✅ Перейди в бота», кнопка «Открыть бота» → `openTelegramLink('https://t.me/CozyReset_bot')`
- [ ] **Result** (из истории) — загрузка по `openResultId`, тот же UI + «Открыть бота»
- [ ] **History** — список сохранённых результатов, «Смотреть результат» → Result с `openResultId`

### 4. Каталог тестов (из спека, раздел 4.2)
Должны быть ровно 7 тестов с глубокими опросами (15–25 вопросов) и коротким описанием:

| № | Название | Вопросов |
|---|----------|----------|
| 1 | Тревога и беспокойство | 22 |
| 2 | Настроение и энергия | 20 |
| 3 | Самооценка и внутренний критик | 24 |
| 4 | Эмоциональное выгорание | 25 |
| 5 | Границы и отношения с собой | 18 |
| 6 | Смысл жизни и удовлетворённость | 21 |
| 7 | Прогресс терапии (входной/выходной) | 16 |

Файл с тестами: `src/data/tests.ts`.

### 5. API и бэкенд
- [ ] Базовый URL: `VITE_BOT_BACKEND_URL` или `window.location.origin`
- [ ] `POST /mini-app/init` — уже есть в боте
- [ ] `POST /mini-app/save-test-result` — тело: testId, testTitle, answers, dimensions?, completedAt; ответ: `{ id }`
- [ ] `POST /mini-app/test-history` — ответ: `{ items: [{ id, testId, testTitle, completedAt }] }`
- [ ] `GET /mini-app/test-result/{id}` — ответ: полный объект результата (id, testId, testTitle, answers, dimensions?, completedAt)

Точные форматы запросов/ответов — в `README.md` в корне проекта.

### 6. Сборка и запуск
- [ ] `npm install` → `npm run dev` — приложение открывается
- [ ] `npm run build` — завершается без ошибок (TypeScript и Vite)

### 7. Деплой (после репозитория)
- [ ] Репозиторий: приватный, название `cozyreset-path-to-self-miniapp`
- [ ] Vercel: переменная `VITE_BOT_BACKEND_URL`, build = `npm run build`, output = `dist`
- [ ] Ссылка на Mini App в боте указывает на URL Vercel
